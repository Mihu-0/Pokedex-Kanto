# Pokedex Kanto

Aplicacion Full Stack PERN para administrar Pokemon de Kanto. Permite listar, buscar por nombre, filtrar por tipo, ver detalle, crear, editar y eliminar Pokemon.

Los datos iniciales pueden cargarse desde PokeAPI usando un script de seed del backend. Los tipos se guardan en ingles, como vienen desde PokeAPI, y el frontend los muestra traducidos al espanol.

## Tecnologias

| Frontend | React, Vite, React Router, Axios, Bootstrap |
| Backend | Node.js, Express, PostgreSQL, pg, dotenv, CORS |

## Variables de entorno

### Backend

Crear `backend/.env`:

```env
PORT=3000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=COMPLETAR_NOMBRE_BASE_DE_DATOS
DB_USER=COMPLETAR_USUARIO_POSTGRES
DB_PASSWORD=COMPLETAR_PASSWORD_POSTGRES
```

### Frontend

Crear `frontend/.env` o copiar `frontend/.env.example`:

```env
VITE_API_URL=http://localhost:3000/api
```

Si no se define `VITE_API_URL`, el frontend usa `http://localhost:3000/api`.

## Creacion de la base de datos

Entrar a PostgreSQL y crear la base:

```sql
CREATE DATABASE COMPLETAR_NOMBRE_BASE_DE_DATOS;
```

Ejecutar el schema incluido:

```bash
psql -U COMPLETAR_USUARIO_POSTGRES -d COMPLETAR_NOMBRE_BASE_DE_DATOS -f backend/database/schema.sql
```

Tabla principal:

```sql
CREATE TABLE pokemons (
  id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  pokedex_num INTEGER NOT NULL UNIQUE CHECK (pokedex_num BETWEEN 1 AND 150),
  nombre VARCHAR(100) NOT NULL CHECK (length(trim(nombre)) > 0),
  tipo_1 VARCHAR(30) NOT NULL CHECK (length(trim(tipo_1)) > 0),
  tipo_2 VARCHAR(30),
  altura_m NUMERIC(5, 2) CHECK (altura_m IS NULL OR altura_m >= 0),
  peso_kg NUMERIC(6, 2) CHECK (peso_kg IS NULL OR peso_kg >= 0),
  descripcion TEXT DEFAULT '',
  sprite_url TEXT,
  hp INTEGER CHECK (hp IS NULL OR hp BETWEEN 0 AND 255),
  ataque INTEGER CHECK (ataque IS NULL OR ataque BETWEEN 0 AND 255),
  defensa INTEGER CHECK (defensa IS NULL OR defensa BETWEEN 0 AND 255),
  velocidad INTEGER CHECK (velocidad IS NULL OR velocidad BETWEEN 0 AND 255)
);
```

## Endpoints REST

| Metodo | Endpoint | Descripcion |
|---|---|---|
| GET | `/api/pokemons` | Lista Pokemon. Soporta `nombre`, `tipo`, `page` y `limit` |
| GET | `/api/pokemons/:id` | Obtiene un Pokemon por id |
| POST | `/api/pokemons` | Crea un Pokemon |
| PUT | `/api/pokemons/:id` | Edita un Pokemon |
| DELETE | `/api/pokemons/:id` | Elimina un Pokemon |

## Seed desde PokeAPI

El backend incluye un script extra para cargar los Pokemon del 1 al 150 desde PokeAPI:

```bash
cd backend
node src/scripts/seedPokemons.js
```

Antes de ejecutarlo, la base de datos debe existir, el schema debe estar aplicado y `backend/.env` debe estar configurado.

El script usa `ON CONFLICT (pokedex_num) DO NOTHING`, por lo que evita duplicados por numero de Pokedex.
